# 🚀 Инструкция по деплою (UstaGo)

Это руководство поможет вам развернуть проект на Linux-сервере (Ubuntu/Debian) и поддерживать его в актуальном состоянии.

## 📋 Предварительные требования

1. **VPS Сервер**: Linux (рекомендуется Ubuntu 22.04+).
2. **Инструментарий**: `git`, `curl`, `node`, `npm`, `pm2`.
3. **Менеджер Python**: Рекомендуется использовать `uv` (устанавливается автоматически скриптом `setup.sh`).

---

## ⚡️ Быстрый запуск (Первая установка)

Если вы только купили сервер, выполните эти шаги:

1. Клонируйте проект и перейдите в папку.
2. Дайте права скриптам: `chmod +x scripts/*.sh`.
3. Запустите установку:
   ```bash
   ./scripts/setup.sh
   ```
   *Скрипт установит Postgres, настроит окружение, создаст базу данных и системные службы.*

---

## 🔄 Обслуживание и обновление

Для безопасности мы разделили управление проектом на изолированные скрипты в папке `scripts/`.

### 1. Обновление кода (Deploy)
Запускайте этот скрипт каждый раз после `git push` из локальной среды:
```bash
./scripts/update.sh
```
**Что он делает:**
*   Скачивает свежий код.
*   Обновляет зависимости (`uv sync`).
*   Пересобирает Фронтенд (`npm run build`).
*   Перезапускает API, Админку и Бота.

### 2. Изменение базы данных (Migrations)
Если вы добавили новые поля в базу данных:
```bash
./scripts/migrate.sh
```
*Запускайте только при необходимости. Обновляет схему БД без удаления существующих данных.*

### 3. Управление администраторами
Добавление или обновление пароля админа:
```bash
PYTHONPATH=. uv run python scripts/set_admin.py ЛОГИН ПАРОЛЬ
```

## 🔍 Решение проблем

### ❌ Ошибка 401 в админке
Убедитесь, что вы создали админа через `set_admin.py` прямо на сервере.

### ❌ Фронтенд не видит бэкенд (Network Error)
Проверьте файл `admin_frontend/.env.local` на сервере. Там должен быть прописан правильный IP сервера:
`NEXT_PUBLIC_API_URL=http://ВАШ_IP:8000/api/v1`
