# UstaGo Bot (Marketplace)

Инновационный Telegram-бот для поиска мастеров и управления заказами с мощной панелью управления.

## 🌟 Основные возможности
- **Для заказчиков**: Быстрое создание заявок, выбор категорий, уведомления об откликах.
- **Для мастеров**: Профессиональный профиль, фильтрация заказов по районам и категориям, система баланса.
- **Для админов**: Современный дашборд с аналитикой, модерация пользователей, управление финансами и категориями.

## 🚀 Технологический стек
- **Bot**: Python 3.12+, `aiogram 3.x`
- **Database**: PostgreSQL + `SQLAlchemy` (Async) + `Alembic`
- **Admin Panel**: 
  - Backend: FastAPI
  - Frontend: Next.js + TailwindCSS + Lucide + Recharts
- **Infrastructure**: PM2, Systemd, nginx, `uv`

## 📦 Управление проектом

Все действия по управлению проектом теперь вынесены в папку `scripts/` для безопасности и удобства:

1. **Первичная установка**:
```bash
chmod +x scripts/*.sh
./scripts/setup.sh
```

2. **Обновление кода (деплой)**:
```bash
./scripts/update.sh
```

3. **Миграции базы данных** (если изменилась структура):
```bash
./scripts/migrate.sh
```

## ✅ Команды, которые были выполнены и приняты (с описанием)

### 1. Создание файла `.env.local` с URL бэкенда
```bash
echo "NEXT_PUBLIC_API_URL=http://31.3.209.51:8000/api/v1" > admin_frontend/.env.local
```
**Что делает:**  
Записывает в файл `admin_frontend/.env.local` переменную окружения, которая указывает фронтенду, по какому адресу обращаться к API бэкенда.  
**Почему важно:** Без этого фронтенд пытался бы стучаться на `localhost:8000` (недоступно с компьютера пользователя). Теперь он использует реальный IP сервера.

---

### 2. Запуск скрипта обновления (сборка + перезапуск)
```bash
./scripts/update.sh
```
**Что делает:**  
- Получает последние изменения из Git.  
- Обновляет Python-зависимости.  
- Собирает фронтенд (Next.js) — теперь с подхваченным `.env.local`.  
- Перезапускает оба PM2-процесса (`ustago-backend` и `ustago-frontend`).  
- Перезапускает Telegram-бота.  

**Результат:**  
Сборка прошла успешно (TypeScript без ошибок), оба сервиса `online`, фронтенд видит правильный API URL.

---

### 3. Открытие порта 8000 в фаерволе
```bash
sudo ufw allow 8000/tcp
```
**Что делает:**  
Добавляет правило в `ufw` (Uncomplicated Firewall), разрешающее входящие TCP-соединения на порт 8000.  
**Почему важно:** Бэкенд FastAPI слушает этот порт, и без открытого порта браузер не смог бы отправлять запросы к API.  
(Порт 3000 был открыт ранее, система пропустила добавление как уже существующее правило.)

---

### 4. Установка пароля для администратора
```bash
PYTHONPATH=. uv run python scripts/set_admin.py admin admin123
```
**Что делает:**  
Запускает скрипт, который ищет пользователя с username `admin` в базе данных и устанавливает ему хешированный пароль `admin123`.  
**Результат:**  
Вывод `Successfully set password for admin: admin` — пароль успешно задан. Теперь можно войти в админ-панель с этими данными.

---

### 🔍 Проверка статуса после всех действий

Оба процесса `online`, без перезапусков (столбец `↺` = 0 или 1 — не критично).
